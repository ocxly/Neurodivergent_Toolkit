# Contributing

Welcome! This project encourages small, bite-sized contributions — fix a typo,
improve a doc, add a comment, or suggest a feature.
**Tiny contributions add up, and are deeply appreciated.**

- New here? See the **Quick Start** in the [README](README.md#-quick-start) — there's no build step.
- If you only have energy for one small thing, that's a huge help.
- No contribution is too small: all help makes a difference.

## How to contribute

1. **Fork** the repo and create a branch for your change.
2. Make your edit. Each tool is a single self-contained `index.html`, so you can
   open it directly in a browser to test — no tooling required.
3. **Test what you touched**: open the affected tool and click through it in both
   light and dark themes.
4. Open a **pull request** describing what you changed and why.

## What makes a good change here

Because this toolkit is for neurodivergent users, please keep these in mind:

- Keep pages **self-contained** (inline CSS/JS, no build tooling or dependencies).
- Preserve **accessibility**: `lang` attribute, labels on inputs, visible focus
  outlines, and `prefers-reduced-motion` for anything animated.
- Support **both light and dark** themes using the shared CSS variables.
- Favor **calm, low-clutter** UI and plain language.

## Reporting ideas or bugs

Not up for code? Opening an issue is a real contribution. The
[issue template](ISSUE_TEMPLATE.md) helps you describe what you're trying to do
and whether it's a high- or low-energy task.
